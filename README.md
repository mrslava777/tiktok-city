# 🏰 TikTok City

Мобильное веб-приложение для TikTok Live. На экране — живой **город** с четырьмя
параметрами: **население, еда, казна, защита**. Каждые **30 секунд** ресурсы
убывают. События TikTok Live (лайки, комментарии, подарки, подписки, репосты)
пополняют ресурсы и поднимают **уровень города**.

Интерфейс адаптирован под телефон: крупные полосы прогресса, лог событий и
уровень города. Данные хранятся **только в памяти**, без базы данных — после
перезапуска сервера город начинается заново.

## Стек
- Node.js + Express — сервер и статика
- Socket.IO — обновления в реальном времени
- TikTok Live Connector — события из лайва

## Как запустить
```bash
npm install
npm start
```
Открой на телефоне (в той же сети) или локально: `http://localhost:3000`

Переменная окружения `PORT` задаёт порт (по умолчанию 3000).

## Как играть
1. Введи `@username` аккаунта, который **сейчас в прямом эфире** TikTok, и нажми «Подключить».
2. Смотри, как события стрима меняют ресурсы города в реальном времени.
3. Кнопка «🎲 Тест-событие» имитирует активность — удобно проверить без живого стрима.
4. «🔄 Сброс» — начать город заново.

## Правила (как событие влияет на город)
| Событие TikTok | Эффект |
|---|---|
| ❤️ Лайк | +еда (и чуть населения) |
| 💬 Комментарий | +население |
| 🎁 Подарок | +казна, +защита, +еда (по числу алмазов) |
| ➕ Подписка | +население |
| 🔁 Репост | +защита |
| 👋 Заход зрителя | +население |

Каждые 30 секунд: население −2, еда −4, казна −3, защита −3. Если ресурс на нуле —
в логе появляется предупреждение.

## Структура
- `server.js` — Express + Socket.IO, таймер убывания, REST API
- `game.js` — игровое состояние города и правила событий (в памяти)
- `tiktok.js` — обёртка над TikTok Live Connector
- `public/index.html` — мобильный интерфейс

> Примечание: TikTok Live Connector подключается только к аккаунтам, которые
> прямо сейчас ведут эфир. Если эфира нет — будет ошибка «оффлайн», это нормально.
